The unreasonable effectiveness of HTML

Why we're switching from markdown to HTML for agent-generated artifacts.

The premise

"Markdown flattens spatial information. HTML restores it."

Thariq Shihipar · Anthropic · May 2026

Side-by-side benchmark
17/20

HTML won 17 of 20 head-to-head comparisons against markdown for the same agent output. The three losses were tasks that never get read by a human.

Where HTML wins

Nine kinds of work.

  • 01
    Exploration & planning — fan out options side-by-side, pick one, then plan it.
  • 02
    Code review — annotated diffs, module maps, the PR writeup reviewers want.
  • 03
    Reports — status updates and post-mortems that get read, not skimmed.
  • 04
    Explainers — collapsible steps, tabbed snippets, glossary in the margin.
  • 05
    Custom editors — throwaway UIs for the exact thing you're working on, always with an export button.
What changes for us

Default still markdown — HTML on request.

Stays markdown

Memory files, CLAUDE.md, plans-of-record, commit messages, PR descriptions, Slack/Telegram replies.

Becomes HTML

Status reports, incident write-ups, explainers, slide decks, dashboards, anything spatial.

In short

One file. No build step.
You stay in the loop.

The artifact is a `.html` you open in a browser. The agent generates it; you tweak it; the agent regenerates it. The loop gets tighter.
